💡 企画内容・顧客体験
```text
(背景) 
・人と人とが顔を合わせ、膝を突き合わせて議論をしたり、ものやサービスづくりをしたりとした後に、スッと解散して、また新たな仕事や活動に進みがちな日常。
・でも、チームでの解散時には、たくさんのドラマと刻むべき日々の想いがある。それを紡いで、未来に繋げるきっかけができないか。
・まず、メンバーの想いで、歌・動画・色紙を紡ぐ。それで、チームの軌跡を次なる挑戦の力へと繋ぐ。
・そして想いだけでは、忘れがちになるため、その想いからなるグッズをECサイトで購入可能に。またチームメンバーが手作りでつくったものも、そこにあげることができ、互いに未来に繋ぐものづくりを楽しむプラットフォームに。

(内容) 
・まずは幹事がボードを立ち上げる。すると、参加者用URLと管理者URLが表示される。
・参加者用URLでは、テキストだけか、画像とともに投稿 (画像はストレージ契約をしていないため、アップロードのみでギフトにはまだ反映していない)
・幹事用URLでは、プレビューをした上で、最終化するボタンを実装。幹事が合言葉とURLをチームメンバーに送付したらプレゼントに。
・今回ログインが課題であるが、そのログイン新美をもたせた。ギフトを開封するための、楽しみになるログインという実装を試みた。
・ギフトを開けると、まずメッセージから紡がれる歌詞とBGMが流れる（ここは生成AIを使う予定）
・そして次に進むと、生成AIで紡がれた動画が流れる
・そして次に進むと、ECサイトでグッズが手に入る
```

🏋️ 工夫した点・こだわった点
```text
・世界中でも珍しいログイン機能が楽しみになるコンテンツ
・色紙や動画などの単体サービスはあるが、それを顧客のために統合されるギフトサービスは少ない
・ECサイトにまでなることで、ビジネス機会にもなるのでは。
```

🗻 難しかった点・次回トライしたいこと（又は機能）
```text
・とにかく過去最大数のファイルをつくり、繋げる作業に苦戦した。AIを活用しながら進めたが、ファイルの構造を理解しながら進めないとうまくいかず、何度もいったりきたりした。
・本番環境で、重要ファイルをwww内にのせない方法を試した。やってみると、ローカル環境とさくらサーバー環境でのファイル位置が変わるために、調整に苦労した。またローカル開発環境は維持をするコードにすることを工夫した。
・デザインをシンプルにすることで、まずミニマルに進めて、後で最低限のデザインで工夫をする開発を進めた。今までよりも使いやすさのために、シンプルにできたのではと思う。
・生成AIが有料ばかりで、どう無料のを使うのか、またストレージをどうするのか、今後学びたい。
・セキュリティ面も、先生に教わったHashを用いてみたが、最初からHashをしようとすると、秘密の合言葉にHashがかかってしまい、後から修正するのに苦労した。Hashは後で実装でも良いかと思った。
```
